# Gitlogue : transformez votre historique Git en une expérience visuelle cinématographique

## TL;DR

- Gitlogue est un outil construit en Rust qui anime visuellement vos commits Git dans le terminal.
- Il simule de manière réaliste les frappes, mouvements de curseur et modifications de fichiers.
- Idéal pour l'onboarding, les démonstrations et les présentations visuelles.
- Personnalisation de thèmes, vitesses de saisie et plus encore.
- Simplifie la compréhension de l'évolution du code et dynamise les workflows Git.

## Un outil qui révolutionne le terminal

Les outils traditionnels pour analyser l’historique des commits Git restent souvent austères. Gitlogue change cette approche en proposant une véritable transformation visuelle : il traduit l’évolution de vos projets en une narration animée et immersive directement dans le terminal. 

Créé avec le langage Rust, Gitlogue s'appuie sur Tree-sitter pour un surlignage syntaxique précis. Plutôt que d’utiliser une ligne de commande classique pour observer les changements dans votre dépôt, Gitlogue fournit des « replays », simulant les mouvements de frappes et les mises à jour des fichiers au sein du projet. C'est une révolution dans la manière d’interagir et de visualiser l’histoire Git.

## Fonctionnalités principales de Gitlogue

### Une expérience cinématographique dans votre terminal

Gitlogue enrichit l’expérience utilisateur en transformant les commits en animations captivantes. Voici les fonctionnalités phares :

- **Animation réaliste de frappe et du mouvement du curseur** : revivez le processus de codage comme si vous regardiez un développeur à l’œuvre.
- **Visualisation des structures de projet** : affiche en temps réel les mises à jour des fichiers, les ajouts ou suppressions et les mouvements dans l’arborescence.
- Support pour les différents états des fichiers : simplifiez la gestion des fichiers en attente, en « staging » ou non suivis.

### Une valeur ajoutée significative pour les développeurs

Gitlogue s’avère utile dans de nombreuses situations :

- **Onboarding des nouveaux développeurs** : Montrez l’évolution d’un module complexe ou l’historique de modifications d’une fonctionnalité critique.
- **Démos dynamiques** : Que ce soit pour des conférences, des vidéos éducatives ou des présentations de projet, Gitlogue capte l’attention grâce à ses visualisations captivantes.
- **Mode amusant** : Un « Look Busy Mode » pour simuler une activité intense pendant des appels ou réunions.

Ces fonctionnalités rendent Gitlogue particulièrement attrayant pour des usages variés, qu’ils soient pédagogiques ou pratiques.

## Cas d'utilisation pratique

Gitlogue n'est pas seulement un outil technique, il devient une ressource indispensable dans plusieurs scénarios :

1. **Dynamiser les présentations de code** : Lors de conférences ou de vidéos techniques, l’utilisation de Gitlogue transforme un simple historique Git en une illustration engageante.
2. **Simplifier les explications complexes** : Que vous écriviez un tutoriel ou formiez une équipe, Gitlogue permet de rendre visuels les concepts souvent abstraits.
3. **Création de contenu pour les développeurs** : Blogueurs techniques, éducateurs ou créateurs vidéo peuvent profiter de Gitlogue pour enrichir leurs publications ou tutoriels.

En exploitant des fonctionnalités avancées, les développeurs disposent d’un moyen innovant pour communiquer leurs idées ou workflows.

## Caractéristiques techniques de Gitlogue

Sous le capot, Gitlogue est conçu pour offrir une expérience fluide et légère aux utilisateurs :

- **Développement avec Rust** : un langage reconnu pour ses performances et sa sécurité, garantissant des animations fluides et un build rapide.
- **Tree-sitter** : permet un surlignage syntaxique précis dans plus de 30 langages, ce qui est essentiel pour une expérience fidèle dans des projets multilingues.
- **Personnalisation poussée** : jouez avec les thèmes visuels, ajustez la vitesse de frappe et configurez vos préférences pour créer un replay qui s’adapte à vos besoins.
- **Flexibilité dans les replays** : explorez l’historique complet d’une série de commits ou concentrez-vous sur un seul.

Les choix techniques derrière Gitlogue assurent qu’il reste performant même lors de la manipulation d’historiques complexes.
