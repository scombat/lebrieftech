# GitHub Actions : Tarification Simplifiée et Nouveaux Frais pour les Runners Auto-hébergés en 2026

## TL;DR

- Les runners hébergés par GitHub bénéficieront d'une réduction de coûts atteignant 39 %, selon le type de machine.
- Un coût de 0,002 $/minute sera appliqué à partir de 2026 pour les runners auto-hébergés utilisés avec des dépôts privés.
- Les dépôts publics resteront gratuits et les clients GitHub Enterprise Server ne seront pas concernés par ces changements.
- Des améliorations significatives sont prévues pour les runners auto-hébergés : meilleure gestion du scaling et support étendu pour Windows.

## Quelles sont les modifications tarifaires pour GitHub Actions en 2026 ?

À compter de 2026, GitHub apporte des ajustements majeurs à la tarification de ses GitHub Actions pour offrir plus de clarté et d’économies aux utilisateurs.

### Diminution des prix pour les runners hébergés par GitHub

À partir du 1er janvier 2026, les coûts associés aux runners hébergés par GitHub baisseront jusqu’à 39 %, selon le type de machine. Les quotas de minutes gratuits fournis avec les plans GitHub ne subiront pas de modifications. Cet ajustement permettra aux équipes et entreprises utilisant les runners hébergés de mieux contrôler leurs coûts, en particulier lors de l’exécution de workflows CI/CD très consommatifs.

### Introduction d'un coût pour les runners auto-hébergés

Une modification plus notable affectera les utilisateurs de runners auto-hébergés. À partir du 1er mars 2026, GitHub introduira un coût de 0,002 $/minute pour les runners utilisés avec des dépôts privés. Cependant, ce coût ne concernera ni les dépôts publics ni les utilisateurs du plan GitHub Enterprise Server. Cette initiative vise à encourager une utilisation plus réfléchie des ressources tout en diversifiant les sources de revenus pour GitHub.

## Quels avantages pour les runners hébergés par GitHub ?

Le choix des runners hébergés par GitHub offre une alternative clé en main pour les équipes ayant peu de ressources à allouer à la gestion de l’infrastructure CI/CD. Avec cette réduction allant jusqu’à 39 %, GitHub renforce l'accessibilité de ce service, le rendant plus compétitif parmi les différentes solutions d'intégration continue disponibles. Les utilisateurs de GitHub Actions pourront continuer à profiter de quotas gratuits pour démarrer rapidement leurs projets, avec un modèle de coûts désormais plus simple à anticiper.

Pour les organisations ayant des workflows complexes, ce changement constitue une opportunité significative de réduire leurs dépenses globales liées à l’automatisation.

## Quelles implications pour les runners auto-hébergés à partir de mars 2026 ?

### Présentation des frais pour les minutes utilisées au-delà des quotas

Les runners auto-hébergés, bien qu'en grande partie gérés par leurs utilisateurs, généreront désormais un coût variable pour les minutes utilisées en dehors des quotas inclus dans les différents plans payants. À 0,002 $ par minute, ce coût supplémentaire restera relativement faible, mais il pourrait avoir un impact pour des équipes ou projets dont les budgets sont serrés, notamment dans les cercles open source utilisant des dépôts privés.

### Pas de frais pour les dépôts publics

Les projets open source bénéficiant de dépôts publics continueront de profiter d’une exécution gratuite des workflows sur des runners auto-hébergés. Cette exonération vise à alléger la charge pour des initiatives souvent essentielles à la communauté de développement.

## Investissements prévus pour améliorer les runners auto-hébergés

En parallèle de ces ajustements tarifaires, GitHub prévoit des améliorations techniques importantes pour les runners auto-hébergés, offrant ainsi de nouvelles possibilités de gestion d'infrastructure pour les utilisateurs.

### Extension des capacités de scaling

GitHub s'engage à gérer plus efficacement les besoins de scaling des runners, en leur permettant notamment d’aller au-delà des conteneurs Linux. De nouvelles méthodes de scaling seront également introduites, offrant une flexibilité accrue pour s'adapter aux charges de travail changeantes.

### Support étendu pour Windows

En réponse aux besoins croissants des développeurs travaillant dans des environnements Windows, des capacités spécifiques seront ajoutées, y compris la prise en charge de workflows Windows plus complexes. Cela devrait également inclure une meilleure compatibilité avec les conteneurs Windows, facilitant le déploiement de projets exigeants une couche de virtualisation adaptée.

### Déploiement progressif tout au long de 2026

Ces nouvelles fonctionnalités seront déployées progressivement au cours de l’année 2026, permettant aux utilisateurs d’explorer et d’adopter ces solutions à leur rythme tout en profitant d'un support continu.

## À retenir

Les changements annoncés pour 2026 représentent une évolution significative dans la manière dont GitHub Actions est tarifé et intégré :

- Les runners hébergés par GitHub deviendront plus abordables avec des réductions allant jusqu’à 39 %.
- Les runners auto-hébergés auront un coût modique de 0,002 $/minute pour les dépôts privés, sans effet sur les dépôts publics ni sur les clients GitHub Enterprise Server.
- Des améliorations techniques substantielles autour du scaling et du support pour Windows offriront de nouveaux bénéfices aux développeurs.

Pour les équipes et individus utilisant des workflows intensifs, ces ajustements présenteront à la fois des opportunités de réduction des coûts et des besoins d’adaptation stratégique.
