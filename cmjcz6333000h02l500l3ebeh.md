# Les acteurs étatiques exploitent les codes OAuth pour accéder aux comptes Microsoft 365

## TL;DR

- Les flux OAuth 2.0 de Microsoft sont détournés par des cybercriminels pour accéder illégalement aux comptes Microsoft 365.
- Les attaques reposent sur des emails de phishing qui incitent les victimes à saisir des codes OAuth légitimes sur des domaines Microsoft vérifiés.
- Ce vecteur de menace contourne les défenses traditionnelles : aucun mot de passe volé, aucune authentification multi-facteurs compromise.

## Qu'est-ce que le code OAuth ?

OAuth 2.0 est un protocole d'authentification standardisé largement utilisé pour accéder à des services en ligne sans partager directement les identifiants. À l'aide d'un code d'appareil, l'utilisateur autorise une application ou un service à accéder à son compte après avoir confirmé son identité. Contrairement aux méthodes classiques d'authentification, le flux OAuth utilise des jetons temporaires générés par des serveurs de confiance, offrant ainsi une alternative pratique et sécurisée aux mots de passe.

Cependant, ce mécanisme présente une faille critique dans le contexte des attaques : les codes d'appareil peuvent être manipulés si les utilisateurs sont incités, via des techniques de phishing, à les soumettre sur des pages légitimes. Les jetons ainsi obtenus offrent aux acteurs malveillants un accès direct aux comptes ciblés.

## Comment se déroulent ces attaques ?

### Étapes principales de l’exploitation

1. **Première phase : hameçonnage**  
   Les attaquants identifient leurs cibles, souvent par le biais d'emails frauduleux contenant des liens ou des QR codes. Ces messages semblent provenir de sources légitimes et incitent les victimes à agir rapidement.

2. **Seconde phase : saisie du code d’appareil**  
   La victime est redirigée vers une véritable page Microsoft où elle est invitée à saisir un code d'appareil. Cette étape est essentielle, car elle se déroule dans un environnement de confiance, réduisant les soupçons.

3. **Finalisation : obtention des jetons d’accès**  
   Une fois le code validé, les attaquants génèrent des jetons OAuth permettant un accès complet aux comptes Microsoft 365. Ces jetons contournent les systèmes de sécurité classiques et ne déclenchent aucune alerte notable.

### Pourquoi cette méthode est-elle efficace ?

Contrairement aux attaques traditionnelles impliquant le vol de mots de passe ou le contournement des mécanismes d'authentification multi-facteurs (MFA), l'approche utilisant les codes d'appareil profite de la légitimité des flux OAuth. Les systèmes de sécurité ne perçoivent pas ces interactions comme suspectes, ce qui en fait un vecteur particulièrement discret.

## Quels sont les groupes impliqués ?

Divers groupes de cybercriminels exploitent ces techniques, incluant des entités financées par des gouvernements :

- **TA2723**  
   Groupe motivé par des intérêts financiers, principalement orienté vers l'espionnage d'entreprise et le vol de données sensibles.

- **UNK_AcademicFlare**  
   Associé à des activités étatiques russes, ce groupe cible les infrastructures critiques, notamment militaires et gouvernementales.  

- **Groupes chinois et autres acteurs malveillants**  
   Plusieurs entités non confirmées exploitent également les flux OAuth, généralement à des fins d'espionnage industriel ou militaire.

Ces organisations s’appuient souvent sur des frameworks avancés tels que **SquarePhish 2** et **Graphish** pour automatiser les campagnes d’attaque. Ces outils abaissent les barrières techniques, permettant même à des groupes sans expertise significative de mener des exploits complexes.

## Les impacts pour les utilisateurs et entreprises

### Risque pour les utilisateurs finaux

Pour les victimes individuelles, ces intrusions permettent aux attaquants d'accéder à des informations personnelles, professionnelles ou financières. Cela inclut la compromission d'emails confidentiels, de fichiers stockés dans Microsoft OneDrive ou de calendriers professionnels.

### Menace pour les entreprises

Au niveau des organisations, le détournement de comptes Microsoft 365 peut entraîner :

- De graves violations de données sensibles.
- Des interruptions opérationnelles dues à l'utilisation malveillante des jetons d’accès.
- Une atteinte à la réputation professionnelle en cas de divulgation non autorisée de documents ou d'informations confidentielles.

Ces risques combinés soulignent la nécessité de renforcer les dispositifs de sécurité liés aux flux OAuth.

## Recommandations et mesures préventives

Pour contrer ces attaques, plusieurs actions sont préconisées :

### Stratégies techniques

- **Blocage des flux OAuth 2.0 pour appareils**  
   Proofpoint recommande aux entreprises de désactiver ces flux à travers des politiques d'accès conditionnel. Cela empêche leur exploitation par des attaquants.

- **Adoption de méthodes "passwordless" et FIDO**  
   Les solutions basées sur l’authentification biométrique ou les clés matérielles sont plus robustes face aux attaques ciblant les codes OAuth.

### Sensibilisation des utilisateurs

Éduquer les membres d'une organisation sur les risques liés aux pages d'authentification et aux emails frauduleux. Les utilisateurs doivent être en mesure de reconnaître les tentatives d'hameçonnage et adopter une posture prudente devant toute demande de code ou de QR code.

### Surveillance des nouvelles menaces

Les outils tels que SquarePhish et Graphish évoluent rapidement, élargissant leurs capacités d'exploitation. Pour se tenir informées, les entreprises doivent collaborer avec des spécialistes en cybersécurité et surveiller les publications d'acteurs majeurs comme Microsoft ou Proofpoint.

## Conclusion

Le détournement des flux OAuth 2.0 représente une menace critique pour les utilisateurs et entreprises de l’écosystème Microsoft 365. En combinant sensibilisation et restriction des flux techniques, il est possible de réduire ces risques. Les organisations doivent également préparer leurs défenses face à une diffusion accrue de ces techniques parmi les groupes moins spécialisés.
