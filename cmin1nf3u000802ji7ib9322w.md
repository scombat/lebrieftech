# Linux Kernel 6.18 : tout ce qu'il faut savoir sur cette version majeure

## TL;DR

- Améliorations significatives pour le matériel : CPU, GPU, stockage, et réseau.
- Nouvelle version potentiellement candidates pour devenir LTS.
- Introduction du pilote Rust Tyr pour GPU ARM Mali.
- Optimisations des systèmes de fichiers comme Btrfs et XFS.
- Installation recommandée pour utilisateurs avancés seulement.

## Les points forts de Linux Kernel 6.18

Linux Kernel 6.18, publié en 2025, apporte des avancées notables dans plusieurs domaines clés, notamment l'optimisation matérielle et des fonctionnalités destinées aux développeurs et ingénieurs. Cette version marque une étape significative, avec des améliorations pour les processeurs Intel et AMD, des ajouts au Device Tree, une gestion affinée des systèmes de fichiers, et le lancement d'un pilote expérimental pour les GPU ARM Mali.

### Améliorations pour Intel

Les utilisateurs équipés de matériel Intel profiteront de nouveautés notables :

- **Support d'affichage** pour les processeurs Wildcat Lake.
- Gestion énergétique améliorée avec le **Panther Lake SoC Power Slider**, visant à équilibrer performance et consommation d'énergie.
- Mise à jour du pilote **P-State**, renforçant les capacités d'optimisation énergétique.
- Compatibilité accrue avec la technologie **Intel TDX**, ajoutant davantage de sécurité aux infrastructures basées sur des machines virtuelles.

### Nouveautés au Device Tree

Le Device Tree s'enrichit pour supporter un éventail de nouvelles configurations matérielles. Parmi les améliorations :

- Intégration des puces Apple **M2 Pro/Max** grâce au projet **Asahi Linux**, permettant une meilleure compatibilité pour les ordinateurs Mac de dernière génération.
- Support étendu pour les laptops Snapdragon X1 tels que certains modèles Inspiron 14 Plus.
- Premières contributions pour les processeurs basés sur RISC-V, notamment le **SiFive HiFive Premier P550**.

### Refinements pour AMD

AMD bénéficie également d’ajustements importants dans cette version, notamment pour ses processeurs et environnements virtualisés :

- Support des processeurs serveur **EPYC Venice**, avec des améliorations pour le stockage mémoire.
- Correction majeure pour les configurations virtualisées dépassant 255 vCPUs.
- Sécurisation renforcée grâce à la prise en charge des **machines virtuelles SEV-SNP**, garantissant une isolation optimale dans les environnements cloud.

### Pilote Tyr pour GPU ARM Mali

Le noyau introduit un **pilote Rust expérimental**, nommé Tyr, pour les GPU Mali ARM. Destiné à remplacer l'ancien pilote Panthor, ce nouvel ajout met l'accent sur la modernisation des outils de gestion GPU. Bien qu'il soit encore en phase expérimentale, Tyr représente une avancée prometteuse pour les développeurs utilisant des solutions graphiques ARM.

### Optimisations des systèmes de fichiers et stockage

La version 6.18 se focalise également sur la performance des systèmes de fichiers et la gestion du stockage :

- Disparition du système de fichiers **Bcachefs** du noyau principal : il restera accessible uniquement via DKMS pour les utilisateurs qui souhaitent l'intégrer manuellement.
- Amélioration des performances d'accès et de lecture pour les systèmes **Btrfs** et **XFS**, avec une optimisation notable des temps de latence.
  
Ces ajustements ciblent les environnements nécessitant une grande réactivité et des capacités accrues en gestion des données.

### Autres ajouts divers

Outre les améliorations matérielles et de systèmes de fichiers, le noyau introduit des fonctionnalités supplémentaires :

- **Support initial pour les pavés tactiles haptiques**, offrant une meilleure expérience utilisateur pour les ordinateurs portables compatibles.
- **Optimisation des performances UDP**, particulièrement efficace en cas de trafic réseau élevé.

## Comment installer Linux Kernel 6.18

Pour les distributions **rolling release** comme Arch Linux ou Fedora, la version devrait être disponible prochainement via les dépôts. Néanmoins, il est conseillé aux utilisateurs moins expérimentés d'attendre les mises à jour officielles des distributions, afin d'éviter des risques de dysfonctionnement.

Pour ceux qui souhaitent procéder à une installation manuelle, le noyau peut être téléchargé directement depuis le site officiel : [Kernel.org](https://www.kernel.org/). Précaution et expertise sont toutefois nécessaires, car certaines fonctionnalités de la version 6.18 restent expérimentales et peuvent présenter des incompatibilités.

Avec des améliorations variées pour le support matériel et les systèmes critiques, elle s'avère particulièrement prometteuse. Les utilisateurs avancés peuvent dès à présent explorer ces nouveautés et contribuer à l'évolution du noyau.
