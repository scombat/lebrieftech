# AWS Control Tower : une expérience dédiée aux contrôles

## Qu'est-ce qu'AWS Control Tower ?

AWS Control Tower est une solution proposée par Amazon Web Services pour simplifier la gestion et la gouvernance des environnements cloud multi-comptes. Elle aide les organisations à mettre en place des règles de conformité, des politiques de sécurité et des outils de gestion de manière centralisée, tout en offrant une architecture réplicable à grande échelle.

Traditionnellement, les entreprises utilisant AWS Control Tower doivent configurer une "landing zone" pour accéder aux fonctionnalités avancées de contrôle et de gouvernance. Cette étape, bien que efficace pour les nouveaux environnements, peut s’avérer complexe et chronophage pour les organisations qui possèdent déjà une infrastructure multi-comptes en production. 

## Nouvelle expérience dédiée aux contrôles

AWS introduit une nouvelle fonctionnalité au sein de Control Tower : l’expérience dédiée aux contrôles. Cette option permet aux entreprises de profiter directement des contrôles gérés par AWS sans avoir à configurer toute l'infrastructure d'une landing zone. Concrètement, cela signifie que les organisations qui disposent déjà d'une architecture multi-comptes AWS peuvent accéder aux outils de gouvernance sans devoir réinitialiser ou transformer leur environnement actuel.

Cette expérience répond à une demande croissante d'entreprises souhaitant renforcer leur gouvernance sans investir dans une refonte complète de leur infrastructure cloud. Elle s'oriente vers une approche simplifiée, rapide et directement opérationnelle.

### Ce qui change :

- Pas besoin de configurer un AWS landing zone complet.
- Accès immédiat au catalogue des contrôles gérés par AWS.
- Réduction de la complexité opérationnelle pour les environnements existants.

## Avantages de cette fonctionnalité

L'expérience dédiée aux contrôles d'AWS Control Tower offre plusieurs avantages notables pour les développeurs et ingénieurs en charge des infrastructures cloud :

- **Accès rapide à la gouvernance** : Grâce à cette option, les entreprises peuvent immédiatement commencer à appliquer les contrôles gérés sans étape d’initialisation complexe.
- **Flexibilité** : Idéal pour les entreprises disposant déjà d’un environnement multi-comptes, elles n’ont pas besoin de modifier leur architecture existante.
- **Économie de temps et de ressources** : L’intégration rapide évite les longs processus de déploiement avec des configurations personnalisées.
- **Conformité accrue** : Les contrôles gérés permettent d’améliorer la sécurité et la conformité de l’infrastructure, avec une gestion centralisée basée sur les meilleures pratiques AWS.

## Comment activer la fonctionnalité

Activer l’expérience dédiée aux contrôles d’AWS Control Tower est simple et rapide. Voici les étapes à suivre :

1. **Accéder à AWS Control Tower** :
   Dirigez-vous vers la page d’accueil AWS Control Tower depuis la console AWS.

2. **Choisir l'option d'activation** :
   Sélectionnez **Activer AWS Control Tower**. Deux configurations vous seront proposées :
   - Créer un environnement complet (landing zone).
   - Utiliser uniquement les contrôles gérés AWS pour un environnement existant.

3. **Configurer la région d'accueil** :
   Définissez la région AWS qui servira de base pour le provisionnement des ressources nécessaires.

4. **Activer l’enrôlement automatique** :
   Si désiré, activez l’option permettant l’ajout automatique des nouveaux comptes au sein de l’unité organisationnelle.

5. **Finaliser la configuration** :
   Cliquez sur **Activer AWS Control Tower** pour démarrer le processus.

### Ce qu’AWS configure pour vous :

Une fois activé, AWS Control Tower met en place les éléments suivants :
- Des rôles IAM indispensables à la gestion des contrôles.
- Un enregistreur Config lié au service, destiné à configurer les contrôles de détection et prévenir les violations de conformité.
- Un accès facilité à la gestion des unités organisationnelles, des comptes partagés et des politiques de contrôle (préventif et de détection).

## Quels contrôles sont inclus ?

La nouvelle expérience dédiée inclut les deux principaux types de contrôles suivant :

- **Contrôles préventifs** : Ils assurent le respect des politiques organisationnelles dès le déploiement des ressources. Par exemple, ils peuvent interdire la création de ressources dans des régions spécifiques ou forcer l’utilisation du chiffrement.
  
- **Contrôles de détection** : Ces contrôles identifient rapidement les violations potentielles de configuration au sein des ressources AWS. Les violations détectées permettent de réagir efficacement pour rectifier les erreurs ou renforcer la sécurité.

Pour permettre une gestion personnalisée, AWS offre également la possibilité d’ajouter des enregistreurs Config supplémentaires afin d’optimiser les coûts liés au service AWS Config tout en répondant aux besoins spécifiques de l’organisation.

## Disponibilité et ressources supplémentaires

Bonne nouvelle pour les entreprises utilisant AWS Control Tower : l’expérience dédiée aux contrôles est disponible **dans toutes les régions AWS** prenant en charge Control Tower. Vous pouvez y accéder dès aujourd’hui et commencer à simplifier votre gouvernance cloud.
